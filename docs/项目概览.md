# 项目概览

## 项目定位

`apex-agent` 是一个智能体运行时后端，核心职责不是“只接一个模型接口”，而是把一轮完整的 Agent 执行链路组织起来，包括：

- 接收用户会话请求并维护会话上下文
- 在不同阶段限制或开放不同工具
- 在 `ReAct` 与 `PlanExecutor` 两种执行模式之间切换
- 统一管理 MCP、SubAgent 与 Skills
- 在执行前后做记忆召回、摘要压缩、长期记忆抽取与管理
- 通过 SSE 按统一协议向前端持续推送事件流

## 核心能力

### 1. 多阶段 Agent 引擎

项目把一次会话拆成三个内部阶段：

- `THINKING`：理解用户意图，只允许激活技能或直接作答
- `MODE_CONFIRMATION`：选择进入 `ReAct` 还是 `PlanExecutor`
- `EXECUTION`：真正执行任务、调用工具并产出结果

### 2. 双执行模式

- `react`：适合步骤较少、可以边观察边行动的任务
- `plan-executor`：适合需要先写计划、再按阶段推进的复杂任务

### 3. 统一工具编排

运行时会把以下工具合并进当前会话：

- 内置工具：`ask_human`、`direct_answer`、`write_mode`、`write_plan_tool`、`update_plan_tool`
- MCP 工具：通过 stdio 拉起外部 MCP Server 并注册为 ToolCallback
- SubAgent 工具：通过 HTTP + SSE 调用其他 Agent
- Skills 工具：从 `SKILL.md` 目录动态生成 `activate_skill`

### 4. Memory 系统

记忆模块不仅保存上下文，还把会话拆成多层能力：

- 会话上下文存储：保存当前会话状态、未压缩消息、摘要边界
- 记忆召回：在每轮开始前召回画像、执行历史、Agent 经验
- 对话压缩：消息过长时生成摘要，控制输入上下文体积
- 记忆抽取：在回合结束或空闲一段时间后抽取长期记忆
- 记忆管理接口：支持分页查看、编辑、删除与清空执行历史

## 技术栈

- `Spring Boot 3.4.5`
- `Spring AI Alibaba Agent Framework 1.1.0.0-RC2`
- `Spring AI MCP Client 1.1.2`
- `DashScope / Qwen`
- `MyBatis-Plus 3.5.5`
- `PostgreSQL`（可选，默认不启用）
- `Lombok`

## 当前源码结构

### 仓库层级

- `README.md`：仓库入口说明
- `docs/`：项目文档
- `apex-agent/`：Maven 工程

### 代码主包

- `org.gemo.apex.web`：SSE 聊天接口
- `org.gemo.apex.core`：`SuperAgent` 执行引擎与上下文工厂
- `org.gemo.apex.component`：工具调用管理器、工具拦截器、注册中心
- `org.gemo.apex.tool`：内置工具与 SubAgent 回调
- `org.gemo.apex.memory`：记忆配置、抽取、召回、持久化、管理接口
- `org.gemo.apex.message`：SSE 消息协议模型
- `org.gemo.apex.service`：Agent workspace 与提示词加载
- `org.gemo.apex.config`：全局配置与 Bean 装配

## 默认运行特征

从当前 `application.yml` 可以看出项目的默认运行特征：

- 服务端口：`18083`
- 默认模型：`qwen-plus`
- 默认记忆存储：`in-memory`
- 默认 Agent：`default_agent`
- 已示例配置的 Agent：`default_agent`、`deer-flow`、`meeting_tool`、`contacts_tool`

需要注意的是，当前配置中的 MCP 与 Skills 使用了本地绝对路径示例，真正运行前需要替换成你自己的路径。
