# 快速开始

## 环境准备

建议先准备以下运行条件：

- `JDK 25`
- `Maven 3.9+`
- DashScope API Key
- `PostgreSQL`（仅在 `apex.memory.store.type=jdbc` 时需要）
- `Node.js`（仅在需要拉起示例 MCP Server 时需要）

## 第一步：检查配置

默认配置文件位于：

- `apex-agent/src/main/resources/application.yml`
- `apex-agent/src/main/resources/application-dev.yml`

你至少需要关注这几项：

### 1. 模型密钥

配置环境变量：

```powershell
$env:DASHSCOPE_API_KEY="your-api-key"
```

### 2. 记忆存储方式

默认值是：

```yaml
apex:
  memory:
    store:
      type: in-memory
```

如果切到 `jdbc`，需要先执行：

- `apex-agent/src/main/resources/db/memory-schema-postgresql.sql`

并配置：

- `APEX_MEMORY_DATASOURCE_URL`
- `APEX_MEMORY_DATASOURCE_USERNAME`
- `APEX_MEMORY_DATASOURCE_PASSWORD`

### 3. MCP 与 Skills 路径

当前示例配置里的 MCP 和 Skills 使用了本地绝对路径，例如：

- `E:\1tmp\meeting\...`

这些路径是示例值，实际运行前需要替换成你自己的路径；如果暂时不需要这些能力，也可以先移除对应配置，避免会话执行时尝试加载不存在的外部资源。

## 第二步：启动服务

在仓库根目录执行：

```bash
cd apex-agent
mvn spring-boot:run
```

默认端口：

- `http://localhost:18083`

## 第三步：验证接口

所有接口都必须带 `X-User-Id`。

### 1. 获取可用 Agent

```bash
curl -H "X-User-Id: demo-user" http://localhost:18083/api/sse/agents
```

### 2. 发起 SSE 会话

```bash
curl -N \
  -H "Content-Type: application/json" \
  -H "X-User-Id: demo-user" \
  -X POST http://localhost:18083/api/sse/chat \
  -d '{
    "sessionId": "session-001",
    "agentKey": "default_agent",
    "query": "帮我总结这个项目的架构",
    "type": "NEW"
  }'
```

返回内容会是一串标准化 SSE 消息，具体格式见 [消息标准](消息标准.md)。

### 3. 恢复 `ask_human` 挂起会话

```json
{
  "sessionId": "session-001",
  "agentKey": "default_agent",
  "type": "HUMAN_RESPONSE",
  "humanResponse": {
    "tool-call-id": "approved"
  }
}
```

## Memory 接口

### 查询记忆

```bash
curl -H "X-User-Id: demo-user" \
  "http://localhost:18083/api/memory/items?type=PROFILE&agentKey=default_agent&page=1&pageSize=20"
```

### 更新记忆

`PATCH /api/memory/items/{memoryType}/{memoryId}?agentKey=default_agent`

请求体：

```json
{
  "content": "更新后的记忆内容"
}
```

### 删除记忆

- `DELETE /api/memory/items/{memoryType}/{memoryId}?agentKey=default_agent`

### 清空执行历史

- `DELETE /api/memory/execution-history?agentKey=default_agent`

## 常见问题

### 请求直接返回 400

先检查是否缺少请求头：

- `X-User-Id`

### 会话执行时报 MCP 或 Skills 相关错误

通常是因为：

- `application.yml` 里的示例绝对路径在本机不存在
- 对应的 Node 进程脚本没有准备好

建议先用最小配置跑通主流程，再逐步打开 MCP 与 Skills。

### 想先只验证主框架

可以先保持：

- `apex.memory.store.type=in-memory`
- 关闭或移除不需要的 `mcps`、`skills`

这样更容易把 SSE 会话主链路先跑起来。
