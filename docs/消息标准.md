# 消息标准

> 本文档基于当前代码实现整理，适用于前端、会话 Hub、父 Agent、子 Agent 之间的消息约定。与旧版文档相比，已对齐当前实际事件名与字段结构。

## 通用格式

所有消息都通过统一 JSON 结构传输：

```json
{
  "event_type": "",
  "context": {},
  "messages": []
}
```

## `context` 常用字段

- `mode`：执行模式，常见值为 `react`、`plan-executor`
- `stage_id`：当前计划阶段 ID，仅在 `plan-executor` 下出现
- `executor`：父 Agent 转发子 Agent 消息时补充的执行者名称
- `content_id`：同一段流式内容的聚合标识

## 当前已实现的事件类型

- `STREAM_THINK`
- `STREAM_CONTENT`
- `PLAN_DECLARED`
- `PLAN_CHANGE`
- `TASK_THINK_DECLARED`
- `TASK_THINK_CHANGE`
- `INVOCATION_DECLARED`
- `INVOCATION_CHANGE`
- `ARTIFACT_DECLARED`
- `ARTIFACT_CHANGE`
- `ASK_HUMAN`
- `END`

当前代码里没有独立的 `ERROR` 事件模型；异常通常通过日志处理，并在 finally 中补发 `END`。

## 事件详解

### `STREAM_THINK`

用于输出思考流内容。

```json
{
  "event_type": "STREAM_THINK",
  "context": {},
  "messages": [
    {
      "content": "正在分析用户意图"
    }
  ]
}
```

### `STREAM_CONTENT`

用于输出最终正文或执行结果流。

```json
{
  "event_type": "STREAM_CONTENT",
  "context": {
    "mode": "react",
    "content_id": "c-001"
  },
  "messages": [
    {
      "content": "这是最终答复的正文片段"
    }
  ]
}
```

### `PLAN_DECLARED`

在 `plan-executor` 模式中由 `write_plan_tool` 发出，表示计划已创建。

```json
{
  "event_type": "PLAN_DECLARED",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1"
  },
  "messages": [
    {
      "stage_id": "stage-1",
      "stage_name": "收集信息",
      "description": "整理项目结构与依赖",
      "status": "PENDING"
    }
  ]
}
```

### `PLAN_CHANGE`

用于回写计划状态或计划结构变更。

```json
{
  "event_type": "PLAN_CHANGE",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1"
  },
  "messages": [
    {
      "change_type": "STATUS_CHANGE",
      "stage_id": "stage-1",
      "status": "RUNNING"
    },
    {
      "change_type": "PLAN_CHANGE",
      "operation": "ADD_STAGE",
      "stage_id": "stage-1",
      "new_stage_id": "stage-2",
      "stage_name": "补充验证",
      "description": "追加配置校验阶段"
    }
  ]
}
```

字段说明：

- `change_type`：`STATUS_CHANGE`、`PLAN_CHANGE`、`TRY_REPLAN`
- `operation`：`ADD_STAGE`、`DELETE_STAGE`、`UPDATE_STAGE`

### `TASK_THINK_DECLARED`

用于声明某个计划阶段的“任务思考槽位”。

```json
{
  "event_type": "TASK_THINK_DECLARED",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1"
  },
  "messages": [
    {
      "task_id": "stage-1",
      "content": "开始分析当前阶段"
    }
  ]
}
```

说明：

- 当前代码中定义了该消息模型，但主执行链路暂未显式发送它

### `TASK_THINK_CHANGE`

用于在 `plan-executor` 的阶段内持续追加思考内容。

```json
{
  "event_type": "TASK_THINK_CHANGE",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1",
    "content_id": "c-002"
  },
  "messages": [
    {
      "change_type": "CONTENT_APPEND",
      "task_id": "stage-1",
      "content": "继续补充当前阶段的执行分析"
    }
  ]
}
```

### `INVOCATION_DECLARED`

用于声明某个工具执行标记。

```json
{
  "event_type": "INVOCATION_DECLARED",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1",
    "executor": "会议室助手"
  },
  "messages": [
    {
      "invocation_id": "invoke-001",
      "name": "正在查询会议室",
      "invocation_type": "search",
      "click_effect": "append",
      "content": "",
      "complete": false,
      "render_type": "markdown"
    }
  ]
}
```

### `INVOCATION_CHANGE`

用于更新某个执行标记的状态或内容。

```json
{
  "event_type": "INVOCATION_CHANGE",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1",
    "executor": "会议室助手"
  },
  "messages": [
    {
      "change_type": "STATUS_CHANGE",
      "invocation_id": "invoke-001",
      "status": "COMPLETE"
    },
    {
      "change_type": "CONTENT_APPEND",
      "invocation_id": "invoke-001",
      "content": "命中 3 条候选结果",
      "render_type": "markdown"
    }
  ]
}
```

### `ARTIFACT_DECLARED`

用于声明某个产物。

```json
{
  "event_type": "ARTIFACT_DECLARED",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-2"
  },
  "messages": [
    {
      "scope": "STAGE",
      "data_type": "markdown",
      "source": "knowledge-base",
      "artifact_id": "artifact-001",
      "artifact_name": "需求总结",
      "artifact_type": "document",
      "content": "产物正文",
      "complete": true
    }
  ]
}
```

### `ARTIFACT_CHANGE`

用于追加产物内容。

```json
{
  "event_type": "ARTIFACT_CHANGE",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-2"
  },
  "messages": [
    {
      "scope": "STAGE",
      "change_type": "CONTENT_APPEND",
      "source": "knowledge-base",
      "artifact_id": "artifact-001",
      "artifact_name": "需求总结",
      "artifact_type": "document",
      "content": "新增一段内容"
    }
  ]
}
```

### `ASK_HUMAN`

当模型需要用户补充信息时发送。

```json
{
  "event_type": "ASK_HUMAN",
  "context": {
    "mode": "react"
  },
  "messages": [
    {
      "input_type": "SINGLE_SELECT",
      "question": "请选择部署环境",
      "description": "该操作会影响后续执行路径",
      "options": [
        {
          "label": "staging",
          "description": "先验证测试环境"
        }
      ],
      "tool_call_id": "tool-call-001"
    }
  ]
}
```

支持的输入类型：

- `TEXT_INPUT`
- `SINGLE_SELECT`
- `CONFIRM`
- `MULTI_SELECT`

### `END`

表示本次 SSE 会话结束。

```json
{
  "event_type": "END",
  "context": {
    "mode": "react"
  },
  "messages": []
}
```

## 父子 Agent 转发约定

当前父 Agent 解析子 Agent SSE 流时，处理逻辑如下：

- `STREAM_CONTENT`：拼接进工具返回结果
- `STREAM_THINK`：仅记录日志，不进入最终结果
- `INVOCATION_*`：补充 `executor`、`mode`、`stage_id` 后向外继续发送
- `PLAN_*`、`TASK_THINK_*`：当前实现中忽略
- `END`：标记子 Agent 流读取结束

这意味着当前消息协议已经覆盖了复杂场景，但父子 Agent 之间真正被“桥接转发”的仍以正文和执行标记为主。
