# 消息标准

> 本文档描述当前后端实际发送的 SSE 事件。前端仍可保留对旧事件类型的兼容定义，但主执行链路不再发送 `STREAM_THINK`。

## 通用格式

所有事件统一使用：

```json
{
  "event_type": "",
  "context": {},
  "messages": []
}
```

## `context` 常用字段

- `mode`：执行模式，取值通常为 `react` 或 `plan-executor`
- `stage_id`：当前计划阶段 ID，仅在 `plan-executor` 下出现
- `executor`：父 Agent 转发子 Agent 调用时标记执行者
- `content_id`：同一段流式内容的聚合标识

## 当前主链路会发送的事件

- `STREAM_CONTENT`
- `PLAN_DECLARED`
- `PLAN_CHANGE`
- `TASK_THINK_CHANGE`
- `INVOCATION_DECLARED`
- `INVOCATION_CHANGE`
- `ARTIFACT_DECLARED`
- `ARTIFACT_CHANGE`
- `ASK_HUMAN`
- `END`

## 事件说明

### `STREAM_CONTENT`

用于输出最终正文或 `react` 模式下的执行结果流。

```json
{
  "event_type": "STREAM_CONTENT",
  "context": {
    "mode": "react",
    "content_id": "c-001"
  },
  "messages": [
    {
      "content": "这是最终答复的正文片段"
    }
  ]
}
```

### `PLAN_DECLARED`

在 `plan-executor` 模式下由 `write_plan_tool` 发出，表示执行计划已经创建。

### `PLAN_CHANGE`

由 `update_plan_tool` 回写阶段状态变化或计划结构调整。

- `change_type`：`STATUS_CHANGE`、`PLAN_CHANGE`、`TRY_REPLAN`
- `operation`：`ADD_STAGE`、`DELETE_STAGE`、`UPDATE_STAGE`

### `TASK_THINK_CHANGE`

用于把 `plan-executor` 当前阶段内的分析内容追加到任务节点。

```json
{
  "event_type": "TASK_THINK_CHANGE",
  "context": {
    "mode": "plan-executor",
    "stage_id": "stage-1",
    "content_id": "c-002"
  },
  "messages": [
    {
      "change_type": "CONTENT_APPEND",
      "task_id": "stage-1",
      "content": "继续补充当前阶段的执行分析"
    }
  ]
}
```

### `INVOCATION_DECLARED` / `INVOCATION_CHANGE`

用于声明和更新工具调用标记，便于前端展示工具执行状态、正文追加和执行者来源。

### `ARTIFACT_DECLARED` / `ARTIFACT_CHANGE`

用于声明和追加结构化产物，既支持阶段级产物，也支持全局产物。

### `ASK_HUMAN`

当模型需要用户补充信息时发送。前端收集输入后，以 `RequestType.HUMAN_RESPONSE` 恢复会话。

### `END`

每轮 SSE 在 `finally` 中都会补发 `END`，用于通知前端关闭本轮流。

## 兼容事件

- `STREAM_THINK`：兼容旧协议保留的事件模型，当前主执行链路不再发送
- `TASK_THINK_DECLARED`：事件模型仍存在，但当前主执行链路未显式发送
