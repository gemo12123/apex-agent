# 消息标准

> 该标准同时对应前端、历史记录管理者（简称Hub）、Agent（包括父Agent和子Agent），各方应按照该标准进行各自工作的开展。

## 格式

```json
{
    "event_type": "",
    "context": {
    },
    "messages": [
    ]
}
```

* event_type：用来定义消息类型，分为如下几类：
  * STREAM_THING | STREAM_CONTENT：思考和正文
  * PLAN_DECLARED | PLAN_CHANGE：计划和计划变更
  * INVOCATION_DECLARED | INVOCATION_CHANGE：执行标记和执行标记变更
  * ARTIFACT | ARTIFACT_CHANGE：产物和产物变更
  * ERROR：异常
  * END：结束
* context：存储上下文的信息，如智能体模型（暂时只有plan-executor，影响Agent返回和界面渲染）、阶段ID、invocation_id等
* messages：消息内容

## 消息类型详解

### STREAM_THING | STREAM_CONTENT

```json
{
    "event_type": "STREAM_THING | STREAM_CONTENT",
    "context": {
        "mode": "plan-executor"
    },
    "messages": [
        {
            "content": "正在分析问题。。。"
        }
    ]
}
```

STREAM_THING和STREAM_CONTENT消息格式一样，前端渲染在会话整体的思考和正文部分

### PLAN_DECLARED

```json
{
    "event_type": "PLAN_DECLARED",
    "context": {
        "mode": "plan-executor"
    },
    "messages": [
        {
            "stage_id": "stage-001",
            "stage_name": "阶段1",
            "description": "阶段1描述",
            "status": "PENDING"
        },
        {
            "stage_id": "stage-002",
            "stage_name": "阶段2",
            "description": "阶段2描述",
            "status": "PENDING"
        }
    ]
}
```

规划定义，每个规划有不同的阶段，阶段有对应的唯一不变的ID、名称、描述、状态。状态包括PENDING、RUNNING、COMPLETED、FAILED

### PLAN_CHANGE

```json
{
    "event_type": "PLAN_CHANGE",
    "context": {
        "mode": "plan-executor"
    },
    "messages": [
        {
            "change_type": "STATUS_CHANGE", // 状态变更
            "stage_id": "stage-001",
            "status": "RUNNING"
        },
        {
            "change_type": "PLAN_CHANGE", // 规划变更
            "operation": "ADD_STAGE | DELETE_STAGE | UPDATE_STAGE",
            "stage_id": "stage-00n", // 阶段ID，DELETE_STAGE、UPDATE_STAGE填入需要操作的阶段ID，ADD_STAGE需要填入上一个Stage的ID
            "stage_name": "阶段n",
            "description": "阶段n描述",
            "status": "PENDING" // PENDING,RUNNING,COMPLETED,FAILED
        },
        {
            "change_type": "TRY_REPLAN", // 尝试规划变更
            "stage_id": "stage-00n" // 阶段ID
        }
    ]
}
```

规划变更，目前存在三种变更场景：

* STATUS_CHANGE：阶段状态变更
* TRY_REPLAN：尝试RePlan，该状态不是必须经过的状态，出现该状态时前端应展示一个尝试重新规划的提示标签
* PLAN_CHANGE：规划变更，目前支持ADD_STAGE、DELETE_STAGE、UPDATE_STAGE，无法对已经完成的阶段进行变更。

### INVOCATION_DECLARED

```json
{
    "event_type": "INVOCATION_DECLARED",
    "context": {
        "mode": "plan-executor",
        "stage_id": "", // 各级Agent提供（如有），上层Agent可能篡改
        "executor": "", // 上层Agent提供
    },
    "messages": [
        {
            "invocation_id": "", // invocation_id,各级Agent提供
            "invocation_name": "正在查询xxxx",
            "invocation_type": "search | think...", // 主要用于图标显示
            "click_effect": "", // 该invocation的点击效果，如无反应、列表展示、追加
            "content": "", // 内容，可为空，用于初始化定义
            "complete": false   // 该阶段是否直接结束，默认false
            // 其他字段，例如：追加文本块的文本类型Markdown、代码块的语言类型
        }
    ]
}
```

invocation定义，Agent生成用于标记一个执行过程的开始，前端基于该消息生成一个标签展示。Agent需要生成唯一的id，Agent也可以直接返回complete=true标识该invocation开始执行并已经结束。

### INVOCATION_CHANGE

```json
{
    "event_type": "INVOCATION_CHANGE",
    "context": {
        "mode": "plan-executor",
        "stage_id": "", // 各级Agent提供（如有），上层Agent可能篡改
        "executor": "" // 上层Agent提供
    },
    "messages": [
        {
            "change_type": "STATUS_CHANGE", // 状态变化
            "invocation_id": "", // 各级Agent提供
            "status": "COMPLETE" // 同PLAN_CHANGE
        }, 
        {
            "change_type": "CONTENT_APPEND", // 内容变化
            "invocation_id": "", // 各级Agent提供
            "content": "xxx"
        }
    ]
}
```

invocation变更，目前存在两种变更场景：

* STATUS_CHANGE：状态变更
* CONTENT_APPEND：内容追加，该状态非必须，仅用于模型流式生成追加到该invocation的侧边栏

### ARTIFACT

```json
{
    "event_type": "ARTIFACT",
    "context": {
        "mode": "plan-executor",
        "stage_id": "" // STAGE必填
    },
    "messages": [
        {
            "scope":"STAGE | GLOBAL",
            "source": "", // 来源，如：知识库
            "artifact_id":"",   // 文件ID
            "artifact_name":"", // 文件名
            "artifact_type":"",  // 文件类型
            "content":"",    // 内容，可能是文字、链接、ID等
            "complete": false   // 该阶段是否直接结束，默认false
        }
    ]
}
```

产物定义，Agent生成执行产物，前端基于该消息生成一个产物放在阶段后或正文后。Agent需要生成唯一的id、产物的作用域等，Agent也可以直接返回complete=true标识该产物已生成结束。

### ARTIFACT_CHANGE

```json
{
    "event_type": "ARTIFACT_CHANGE",
    "context": {
        "mode": "plan-executor",
        "stage_id": "" // STAGE必填
    },
    "messages": [
        {
            "scope":"STAGE | GLOBAL",
            "change_type": "CONTENT_APPEND", // 内容变化
            "source": "", // 来源，如：知识库
            "artifact_id":"",   // 文件ID
            "artifact_name":"", // 文件名
            "artifact_type":"",  // 文件类型
            "content":""    // 内容，可能是文字、链接、ID等
        }
    ]
}
```

产物变更，当前仅有CONTENT_APPEND状态，用来标识该产物应该追加的内容。

### ERROR

```json
{
    "event_type": "ERROR",
    "context": {
        "mode": "plan-executor"
    },
    "messages": [
        {
            "content": "异常原因"
        }
    ]
}
```

异常标识，标识智能体执行异常，无法继续输出

### END

```json
{
    "event_type": "END",
    "context": {
        "mode": "plan-executor"
    },
    "messages": [
    ]
}
```

会话结束标识

## 相关方工作

- 前端：基于标准进行前端的渲染，当前仅需要考虑plan-executor模式即可
- Hub：基于标准进行历史会话的管理
- Agent：
  - 父Agent（如有）：接收子Agent消息，对一些消息进行屏蔽（如子Agent的STREAM_THINK）、变更（如子Agent的ARTIFACT变更作用域为STAGE）、派生（如子Agent的STREAM_CONTENT生成STAGE_ARTIFACT）
  - 子Agent：基于消息格式在运行的过程中生成消息
